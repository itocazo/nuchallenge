### 5.2 ISSUES FOUND

#### BUG-01: Prerequisite checkmark always green (P2 — Cosmetic)
**Location:** Challenge detail page, prerequisites section  
**Expected:** If the prerequisite challenge is not completed (status=failed), the checkmark should be gray/red.  
**Actual:** The checkmark is always green (CheckCircle2 icon) regardless of whether the user has actually completed the prerequisite. The server-side 403 catches it on "Start", but the UI misleads.  
**Impact:** User clicks "Start Challenge", gets a cryptic 403 error instead of seeing upfront that the prereq isn't met.

#### BUG-02: Nav bar shows "0 pts" after earning points (P2 — Stale cache)
**Location:** Top navigation bar, points display  
**Expected:** After completing CH-01 and earning 138 pts, the nav should show "138 pts" or at least update on next navigation.  
**Actual:** Nav bar continued to show "0 pts" across all pages during the session. The profile page correctly showed "1.2k pts" (from prior script runs).  
**Impact:** Users don't get real-time feedback that their points increased.

#### BUG-03: Explorer dashboard shows 0/0/0 despite completed challenges (P2 — Stale stats)
**Location:** Explorer page, status cards (Available / In Progress / Completed / Streak)  
**Expected:** After completing CH-01, the dashboard should show "1 Completed" and update Available count.  
**Actual:** All four counters show 0. The stats likely come from the user record rather than being computed from attempts.  
**Impact:** The dashboard feels broken for active users.

#### BUG-04: "Produces an Asset" badge on CH-01 is misleading (P3 — Content)
**Location:** CH-01 challenge detail  
**Expected:** A quiz with a JSON array submission doesn't "produce an asset" that later challenges depend on.  
**Actual:** CH-01 still has `producesAsset: true` and `assetType: 'document'` from before the conversion to auto-graded. No downstream challenge references CH-01 output.  
**Impact:** Minor — users may expect their CH-01 output to appear somewhere later.

#### BUG-05: Interests section shows empty on profile (P3 — Cosmetic)
**Location:** Profile page, Interests card  
**Expected:** If a user has no interests configured, the section should be hidden (like badges).  
**Actual:** Shows an empty "Interests" card with no content.
